How This Model Works

The problem, in plain words

Given how a crop has behaved in a place over the past four years — its yield, rainfall, pesticide use and temperature — we predict next year's yield (in hectograms per hectare). It is a genuine forecast: the model never sees the future year it is asked to predict.

The data

The FAO district panel: 101 countries × 10 crops, yearly from 1990–2013. Each row carries the year's yield, annual rainfall, pesticide tonnage and average temperature. Every country–crop pair is treated as its own time series.

Turning history into windows

A 4-year sliding window moves along each series. Four consecutive years go in; the fifth year's yield is the target. Only strictly contiguous runs are kept, so the network always sees real, gap-free seasons.

Yr 1Yr 2 Yr 3Yr 4 predict Yr 5

The network architecture

A hybrid design fuses two signals:

  • Temporal branch — two stacked Bidirectional LSTMs (96→48 units) read the window forwards and backwards, capturing how early-window conditions set up later ones.
  • Identity branch — learned embeddings for country and crop encode each one's baseline yield level (a tonne of maize differs wildly by place).
  • The two are concatenated → Dense 128 → 64 → 1 with dropout & batch-norm.

How it was trained

  • Leakage-safe split: train ≤2007, validate 2008–2010, test 2011+. Scalers are fit on the training period only.
  • Huber loss (robust to outlier yields) + Adam optimiser.
  • EarlyStopping restores the best validation weights; ReduceLROnPlateau anneals the learning rate. Seeds fixed for reproducibility.

How does it compare to new_model2?

The honest answer: on raw test scores, new_model2's best model is higher — but the two solve different tasks, so the headline numbers are not directly comparable.

ModelTaskTest R²Within ±20%

new_model2 feeds the model the target year's own rainfall, pesticide and temperature values — it explains the current year given current weather (now-casting). This model is only given the past and must predict a year it has never seen (true forecasting), a strictly harder problem — which is why its R² is lower yet arguably more useful for planning ahead.
